# 2izzo/nihongo · Japanese for English speakers

Interactive lesson packs that take an English speaker from zero Japanese toward functional fluency.
Sister project to [2izzo/studio](https://github.com/2izzo/studio) (music theory) — same single-file-HTML format, different domain.

**Live URL:** https://2izzo.github.io/nihongo/

## Audio

Each kana has a pronunciation MP3 generated from an [ElevenLabs](https://elevenlabs.io) ja-JP voice.
The audio is generated locally and committed to the repo — no API key ever ships to the browser.

To regenerate (or generate for the first time):

```bash
# 1. Park your ElevenLabs API key (one-time):
mkdir -p ~/.rizzo/.secrets && chmod 700 ~/.rizzo/.secrets
echo 'sk_your_elevenlabs_key' > ~/.rizzo/.secrets/elevenlabs-api-key
chmod 600 ~/.rizzo/.secrets/elevenlabs-api-key

# 2. Pick a ja-JP-capable voice from https://elevenlabs.io/app/voice-library
#    Edit scripts/generate-audio.mjs and paste the voice ID into VOICE_ID.

# 3. Run:
node scripts/generate-audio.mjs

# Idempotent: skips files that already exist. Re-runs are free.
```

## Local development

The lessons are static HTML/JS/CSS. No build step.

```bash
# From the repo root, serve locally on port 8000:
python3 -m http.server 8000
# Then open http://localhost:8000/hiragana/
```

Audio files are loaded relative to each lesson page, so they only work when served via HTTP — opening the HTML files directly with `file://` will block audio for security reasons. Use the local server above, or push and visit the github pages URL.

## Architecture

`shared/lesson-engine.js` does all the rendering. Each lesson HTML is a thin wrapper that calls `Lesson.init({...})` with lesson-specific config — title, intro, which kana to display, drill counts, quiz pass thresholds, prev/next links. This keeps the per-lesson code small (~30 lines) and makes Pack 2 (katakana) trivial to build: same engine, different data.

`shared/kana-data.js` is the single source of truth for all 104 kana. Adding a new mnemonic or fixing a romaji edits one file, not five.

State is held in module-local closures. **No localStorage, no sessionStorage** — everything resets on page reload. This matches how github pages serves static content reliably.
